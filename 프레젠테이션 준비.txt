0. contents, ui
-----------------------------------------------------------------------------------
페이지 별로 어떤 기능들을 구현할 건지 설명해드릴 텐데
일단 전체적인 ui는 이렇다
앱은 크게 3가지 기능으로 구현 
(1) 혈당 기록하는 페이지, (2) 기록된 내용과 (3) 추이를 확인하는 페이지
-----------------------------------------------------------------------------------

1. 회원가입
이메일 기반 회원등록 -> 보안 인증 및 이메일 발송 필요 시
아이디 비밀번호 기반 회원등록 -> 단순 회원 등록
이름 및 신체 정보, 연락처, 주소 등

-----------------------------------------------------------------------------------
앱을 사용하려면 먼저 회원 등록을 해야함
사용자가 직접 등록할 수 있도록 함
회원등록 방법으로 2가지 정도 생각해 볼 수 있음 - 이메일 기반 / 단순 아이디 비밀번호 기반
(1) 이메일 기반의 경우 
->> 보안적으로 좀 더 안정적일 수 있으나 구글, 네이버, 카카오 등 이메일 인증 필요
->> 즉 타사이트의 회원가입 절차가 필요함

(2) 단순 아이디/비밀번호 기반
->> 앱에서 자체적으로 아이디/비밀번호 생성 및 가입 후 바로 사용이 가능
->> 단순 회원 등록 개념
-----------------------------------------------------------------------------------

2. Drawer
회원 정보 표시 및 추가 설정 기능
- 로그아웃
-----------------------------------------------------------------------------------
간단한 회원 정보를 표시해줌 ( 이름, 나이, 몸무게 등 신체정보, 병원 방문 일자, 회원 별 특이사항 등)

전체적인 앱 설정 기능을 추가할 수 있음 -> 추후에 자주 바꾸게 될 것 같음
앱 테마 설정, 글꼴 설정, 지정된 상담의에게 다이렉트로 문의 문자 보내기 등
-----------------------------------------------------------------------------------

3. 혈당 기록지 작성
날짜별/시간별/상태별
날짜 표시 -> picker 또는 수기 작성
시간 표시 -> 24시간/am pm
memo(스트레스 상황 등 특정 추가 기록 사항) -> picker 또는 수기 작성
-----------------------------------------------------------------------------------
기록지 작성 절차
(1) 날짜 선택하기 - 달력에서 선택할 수 있음 / 수기 작성 가능
(2) 시간 선택하기 - 스크롤로 죽 내려서 시간 선택할 수 있음 / 당연히 수기 작성 가능
(3) 메모지 - 추가 기록 사항 작성하는 부분(스트레스 상황 등 특정 상황일 때),
               카테고리화가 가능하다면 이 부분도 피커로 구현이 가능
(4) 혈당 측정 - mg/dL 단위 사용, 직접 기입함 
                    블루투스 연동이 가능한 측정 기기가 있을 경우 수기 작성 대신
                    연동된 기기에서 측정 시 바로 측정 정보가 앱으로 넘오오게끔 하는 시스템도 생각해볼 수 있음

(5) 저장 버튼 - 저장 버튼을 눌러야 기록지가 작성됨
                   저장된 내용은 History와 Statistics에 반영됨
-----------------------------------------------------------------------------------

4. 혈당 기록 목록
날짜별로 시간대별 혈당 기록 상새 내용을 목록 형태로 표시
목록의 항목 클릭 시 기록 내용 삭제 및 수정 가능
-----------------------------------------------------------------------------------
날짜별로 작성한 기록지 내용들의 목록을 나타내주는 페이지
스크롤해서 작성한 내용들을 모두 확인할 수 있음
작성 내용들을 수정, 삭제할 수 있음
추가적으로 표시할 내용이 필요할 경우 - 클릭 시 동작하는 추가 페이지 구현해야 함
-----------------------------------------------------------------------------------

스와이프로 날짜 선택

6. 추가적으로 생각해 볼 기능

function : 호출 횟수 요금 200만 건 까지 무료, 실행 시간 요금 3,000건 × $0.0000004 = $0.0012
